  - `id: release`
  - `with:` config-file pointing to `release-please-config.json` and manifest-file pointing to `.release-please-manifest.json`

Follow the existing workflow patterns in this repo:
- Use `actions/checkout@v5` style versioning for consistency
- Use the v4 release-please-action (current stable)
- Keep the workflow minimal — no npm publish step (deferred per REQUIREMENTS.md out-of-scope)
- No `workflow_dispatch` trigger needed — release-please should only run on pushes to main

The workflow does NOT need a checkout step — release-please-action handles its own checkout internally.
