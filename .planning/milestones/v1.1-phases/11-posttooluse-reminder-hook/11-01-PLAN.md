  <name>Task 1: Create dc-context-reminder.js PostToolUse hook</name>
  <files>hooks/dc-context-reminder.js</files>
  <action>
Create `hooks/dc-context-reminder.js` following the exact stdin/output/timeout patterns from `hooks/dc-freshness-check.js`. The hook implements:

1. **Stdin timeout guard** -- 3-second `setTimeout(() => process.exit(0), 3000)` pattern identical to Phase 10 hook.

2. **Tool scoping (defense-in-depth)** -- Check `data.tool_name` against allowlist `["Edit", "Write", "MultiEdit"]`. Exit 0 if not in list. This is belt-and-suspenders with the `matcher` in settings.json.

3. **File path extraction** -- Get `data.tool_input.file_path`. If undefined (edge case for MultiEdit), also check `data.tool_input.edits` array for `edits[0].file_path` as a fallback. Exit 0 if no file path found.

4. **CONTEXT.md proximity detection** -- Check two locations only (per user decision):
   - `path.join(editedDir, 'CONTEXT.md')` (same directory)
   - `path.join(path.dirname(editedDir), 'CONTEXT.md')` (parent directory)
   Use `fs.existsSync`. Take the first match found. Exit 0 if neither exists.

5. **Session-scoped debounce** -- Hash `editedDir` using `crypto.createHash('md5').update(editedDir).digest('hex').slice(0, 8)`. Build debounce file path: `path.join(os.tmpdir(), 'dc-reminder-' + sessionId + '-' + dirHash + '.json')`. If file exists, exit 0 (already reminded). Otherwise create it with `{ reminded: Date.now() }`. Wrap debounce file write in try/catch -- proceed with reminder even if write fails.

6. **Reminder output** -- Build relative path via `path.relative(cwd, contextPath)` where cwd comes from `data.cwd || process.cwd()`. Output message: `CONTEXT.md may need updating: {relPath} — you just edited files in this area`. Use exact output format: `{ hookSpecificOutput: { hookEventName: "PostToolUse", additionalContext: message } }`.

7. **Error handling** -- Wrap ALL logic in try/catch with `process.exit(0)` in catch. Hook must never throw or produce stderr output.

Required modules: `fs`, `path`, `crypto`, `os` (all Node.js built-ins, no external deps).
