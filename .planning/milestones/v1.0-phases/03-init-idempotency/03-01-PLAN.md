- If `.context/` does not exist, say nothing and continue.

**Step 3 (modify):** Add a guard at the top: "If both `.context/MANIFEST.md` and `ARCHITECTURE.md` already exist, skip this step entirely (metadata is not needed)." Otherwise, keep the existing inference chain and AskUserQuestion fallback unchanged.

**Steps 4-9 (modify each):** For each step:
- Remove all "Narrate:" instructions -- no per-step output.
- Add status tracking: after each file/directory operation, instruct Claude to "Record status [path]: [status]" where status is created/skipped/updated.
- Keep all existing logic (existence checks, sentinel detection, append behavior) but express the outcome as a recorded status instead of narration.

Specific status logic per item:
- Step 4 directories (.context/domain/, .context/decisions/, .context/constraints/): Check if directory exists BEFORE mkdir. If exists, record "skipped". If not, create it (with .gitkeep) and record "created". Do NOT show .gitkeep files as separate summary lines.
- Step 4 also handle `.context/MANIFEST.md` existence check for Step 5: this is NOT done here, it stays in Step 5.
- Step 5 MANIFEST.md: Add existence check at top. If `.context/MANIFEST.md` exists, record "skipped" and skip. Otherwise, fill template and record "created".
- Step 6 ARCHITECTURE.md: Already has existence check. Change narration to status recording. Record "skipped" if exists, "created" if written.
- Step 7 AGENTS.md: Already has three-way logic. Change narration to: record "created" (file didn't exist), "skipped" (sentinel present), or "updated" (snippet appended).
- Step 8 CLAUDE.md: Already has three-way logic. Change narration to: record "created" (file didn't exist), "skipped" (@AGENTS.md present), or "updated" (@AGENTS.md appended).
- Step 9 .gitignore: Already has three-way logic. Change narration to: record "created" (file didn't exist), "skipped" (.context.local/ present), or "updated" (.context.local/ appended).

**Step 10 (rewrite):** Replace current commit-prompt-only step with two parts:

Part A -- Summary: Instruct Claude to display the accumulated status list in this format:
```
Domain Context initialized:

  .context/MANIFEST.md     created
  .context/domain/         created
  .context/decisions/      created

  ARCHITECTURE.md          skipped
  AGENTS.md                updated
  CLAUDE.md                skipped
  .gitignore               updated

  5 created, 2 skipped, 1 updated
```
Items appear in the order listed above (8 items total). File paths left-aligned, statuses right-aligned with consistent spacing. Use plain text statuses (no emoji/checkmarks).

Part B -- Commit prompt: If ALL items are "skipped" (0 created, 0 updated), say "Everything is already set up. No changes needed." and do NOT show the commit prompt. Otherwise, show the same AskUserQuestion commit prompt as before.
