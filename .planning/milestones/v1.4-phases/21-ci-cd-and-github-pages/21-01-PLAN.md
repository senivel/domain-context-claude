<task type="auto">
  <name>Task 2: Create GitHub Actions deploy workflow and lychee ignore file</name>
  <files>.github/workflows/deploy-docs.yml, .lycheeignore</files>
  <action>
Create `.github/workflows/deploy-docs.yml` with a three-job workflow (build, check-links, deploy):

**Trigger:** Push to `main` branch with paths filter `docs/**`, plus `workflow_dispatch` for manual runs.

**Permissions:** `contents: read`, `pages: write`, `id-token: write`.

**Concurrency:** Group `pages`, `cancel-in-progress: false`.

**Job 1 - build:**
- `runs-on: ubuntu-latest`
- Step 1: `actions/checkout@v5`
- Step 2: `withastro/action@v5` with `path: ./docs`

**Job 2 - check-links:**
- `needs: build`
- `runs-on: ubuntu-latest`
- Step 1: `actions/checkout@v5` (needed for .lycheeignore)
- Step 2: `actions/download-artifact@v4` with `name: github-pages`, `path: ./dist`
- Step 3: Run `tar -xf ./dist/artifact.tar -C ./dist` to extract built HTML
- Step 4: `lycheeverse/lychee-action@v2` with:
  - `args: '--base https://senivel.github.io/domain-context-claude --no-progress ./dist/**/*.html'`
  - `fail: true`
  - `jobSummary: true`

**Job 3 - deploy:**
- `needs: [build, check-links]`
- `runs-on: ubuntu-latest`
- `environment:` with `name: github-pages` and `url: ${{ steps.deployment.outputs.page_url }}`
- Step 1: `actions/deploy-pages@v4` with `id: deployment`

Also create `.lycheeignore` at project root with entries for commonly rate-limited domains:
```
# Rate-limited social media
https://twitter.com/.*
https://x.com/.*

# LinkedIn blocks crawlers
https://www.linkedin.com/.*
```
