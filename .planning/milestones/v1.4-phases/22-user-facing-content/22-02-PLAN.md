Frontmatter:
- title: "User Guide"
- description: "Complete workflow for managing domain context — from initialization to knowledge extraction."

Body content structure:

**Intro paragraph** (2-3 sentences): Domain context lets you codify business knowledge alongside code. This guide walks through the complete workflow. Link to [Quickstart](/domain-context-claude/getting-started/quickstart/) for initial setup.

**H2 "Initializing a project"** (`dc:init`)
- What it does: creates `.context/` directory structure, scaffolds ARCHITECTURE.md, wires AGENTS.md and CLAUDE.md
- What gets created (brief list of directories/files)
- GSD detection: if `.planning/PROJECT.md` exists, the GSD bridge snippet is added automatically
- Source: README dc:init paragraph + commands/dc/init.md objective

**H2 "Exploring domain context"** (`dc:explore`)
- What it does: shows summary of all entries with freshness status
- Keyword search: `/dc:explore caching` searches entries by keyword
- Drill-in: select an entry to view full content
- Source: README dc:explore paragraph + commands/dc/explore.md objective/description

**H2 "Adding new entries"** (`dc:add`)
- What it does: creates a new domain concept, architecture decision, or constraint from a freeform description
- Three types: domain concept, architecture decision, constraint
- Workflow: describe what you want to add, Claude extracts structure, creates file, updates MANIFEST.md
- Example: `/dc:add We chose PostgreSQL over MongoDB because our data is highly relational`
- Source: README dc:add paragraph + commands/dc/add.md objective

**H2 "Validating integrity"** (`dc:validate`)
- What it does: checks structural integrity and reports issues grouped by type
- Check types: broken links, orphan files, stale entries, missing AGENTS.md imports
- Fix offers: after reporting issues, offers to fix each group interactively
- Source: README dc:validate paragraph + commands/dc/validate.md objective

**H2 "Refreshing stale entries"** (`dc:refresh`)
- What it does: identifies entries with verified dates older than 90 days
- For each stale entry: reads content, searches codebase, assesses accuracy, proposes updates
- Updates verified dates in both MANIFEST.md and the file's inline comment
- Source: README dc:refresh paragraph + commands/dc/refresh.md objective

**H2 "Extracting from GSD"** (`dc:extract`)
- What it does: scans completed GSD phase artifacts, classifies findings, lets you selectively create entries
- Scans: CONTEXT.md, SUMMARY.md, RESEARCH.md, RETROSPECTIVE.md from completed phases
- Classifies into: domain concepts, decisions, constraints
- Workflow: preview proposals with source attribution, accept/reject each
- Source: README dc:extract paragraph + commands/dc/extract.md objective

**H2 "GSD integration"**
- Brief explanation of the bridge pattern (AGENTS.md bridge)
- How it works: dc:init adds domain context pointers to AGENTS.md, CLAUDE.md imports AGENTS.md, GSD agents read CLAUDE.md and see domain context
- After completing GSD phases, run `/dc:extract` to promote knowledge
- No GSD configuration changes needed
- Source: README "GSD Integration" section

**H2 "Passive awareness"**
- Brief section on hooks: session-start freshness warnings and edit-time CONTEXT.md reminders
- These work automatically after installation — no commands needed
- Source: README "What Gets Installed" hooks bullet

Tone: Concise, practical, action-oriented. Short sentences. Use `:::tip` asides for helpful notes. Use `:::note` for important context. Use code blocks for all command invocations and terminal output (not screenshots). All internal links must include `/domain-context-claude/` base path.
