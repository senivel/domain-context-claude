<task type="auto" tdd="true">
  <name>Task 1: Add uninstall mode and success messages</name>
  <files>bin/install.js, tests/install.test.js</files>
  <behavior>
    ## Uninstall Tests
    - Uninstall removes dc-prefixed files from target directory (hooks/dc-*.js, agents/dc-*.md, rules/dc-*.md)
    - Uninstall removes commands/dc/ directory contents
    - Uninstall removes templates/ contents (all dc-owned)
    - Uninstall removes tools/validate-templates.sh
    - Uninstall does NOT remove non-dc files (e.g., hooks/gsd-check-update.js)
    - Uninstall does NOT remove directories themselves (only files)
    - Uninstall removes dc hook entries from settings.json SessionStart and PostToolUse
    - Uninstall preserves non-dc hook entries in settings.json
    - After uninstall, settings.json still exists with hooks object (possibly empty arrays removed)
    - Uninstall prints each removed file path
    - --uninstall without --local targets global ~/.claude/
    - --uninstall --local targets local ./.claude/

    ## Success Message Tests
    - After install, prints message containing "domain-context-cc installed"
    - After install, prints next step mentioning /dc:init
    - After uninstall, prints message containing "domain-context-cc uninstalled"
    - After uninstall, prints count of removed files
  </behavior>
  <action>
    Add to bin/install.js:

    1. **removeDcFiles(targetDir) function:**
       - Iterate INSTALL_MAP entries
       - For each, check if target directory exists
       - If filter defined: read dir, remove matching files with fs.rmSync
       - If no filter (templates/, commands/dc/): read dir, remove all files
       - For commands/dc/: also remove the dc/ subdirectory itself (it's dc-owned)
       - Do NOT remove parent directories (hooks/, agents/, rules/, tools/)
       - Return array of removed file paths
       - Print each removed path: `console.log('  removed:', relativePath)`

    2. **removeHooks(settingsPath) function:**
       - Read settings.json (if missing, nothing to do)
       - For SessionStart and PostToolUse arrays, filter out entries where isDcHook is true
       - Remove empty arrays (clean up)
       - Write back with JSON.stringify(obj, null, 2) + '\n'
       - Return count of removed hook entries

    3. **Update main() uninstall branch:**
       - If --uninstall: call removeDcFiles(targetDir) and removeHooks(settingsPath)
       - Print summary: "Removed N files and M hook entries"
       - Print "domain-context-cc uninstalled successfully"

    4. **Polish install success message (INST-09):**
       - After successful install, print:
         ```
         domain-context-cc installed to {targetDir}

         Next steps:
           1. Start a Claude Code session
           2. Run /dc:init in your project to set up domain context
         ```
       - After successful uninstall, print:
         ```
         domain-context-cc uninstalled from {targetDir}
         Removed {N} files and {M} hook entries
         ```

    Per user decisions:
    - Remove dc-prefixed files only -- leave directories intact
    - --uninstall alone cleans global, --uninstall --local cleans local
    - Leave empty hooks object after removing dc hooks -- don't delete settings.json
    - Print each removed file/hook entry to confirm what was cleaned up
  </action>
  <verify>
    <automated>node --test tests/install.test.js</automated>
  </verify>
  <done>All uninstall tests pass. Success messages print after install and uninstall. Running uninstall after install in temp dir leaves no dc files but preserves non-dc files and settings.json.</done>
</task>
