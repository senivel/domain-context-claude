  <behavior>
    ## Unit Tests (pure functions, no filesystem)

    - isDcHook({ hooks: [{ command: "node /path/dc-freshness-check.js" }] }) -> true
    - isDcHook({ hooks: [{ command: "node .claude/hooks/gsd-check-update.js" }] }) -> false
    - isDcHook({ matcher: "Edit", hooks: [{ command: "node .claude/hooks/dc-context-reminder.js" }] }) -> true

    - mergeHooks(empty settings, dc entries) -> settings with only dc entries
    - mergeHooks(settings with GSD hooks, dc entries) -> GSD hooks preserved, dc hooks appended
    - mergeHooks(settings with existing dc hooks, dc entries) -> old dc hooks replaced (no duplicates), GSD hooks preserved
    - mergeHooks(settings with dc + GSD hooks, dc entries) -> idempotent result (same as single install)

    - getDcHookEntries(targetDir, isLocal=false) -> absolute paths in commands
    - getDcHookEntries(targetDir, isLocal=true) -> relative paths (.claude/hooks/...) in commands
    - getDcHookEntries hook entries include matcher "Edit|Write|MultiEdit" on PostToolUse entry

    - getTargetDir(isLocal=false) -> path ending in .claude under home directory
    - getTargetDir(isLocal=true) -> path ending in .claude under cwd

    - parseArgs(['--local']) -> { isLocal: true, isUninstall: false }
    - parseArgs(['--uninstall']) -> { isLocal: false, isUninstall: true }
    - parseArgs(['--uninstall', '--local']) -> { isLocal: true, isUninstall: true }
    - parseArgs([]) -> { isLocal: false, isUninstall: false }

    ## Integration Tests (temp directories)

    - Global install: creates all expected files under temp home/.claude/
    - Global install: creates settings.json with dc hook entries using absolute paths
    - Local install: creates all expected files under temp project/.claude/
    - Local install: creates settings.json with dc hook entries using relative paths
    - Install preserves existing non-dc hooks in settings.json
    - Re-install is idempotent: running twice produces same settings.json (no duplicate hooks)
    - Install creates settings.json if it does not exist
    - Shebang check: first line of bin/install.js is #!/usr/bin/env node
    - __dirname usage: source files resolved relative to bin/../ not process.cwd()
    - Copied shell scripts (validate-templates.sh) retain executable permission
  </behavior>
  <implementation>
    Replace bin/install.js stub with full implementation. Structure as a single CommonJS file (~200-250 lines):

    1. **Constants at top:**
       - INSTALL_MAP array: source-to-dest directory mappings with optional `filter` functions
         - { src: 'commands/dc', dest: 'commands/dc' } (all files)
         - { src: 'hooks', dest: 'hooks', filter: f => f.startsWith('dc-') }
         - { src: 'agents', dest: 'agents', filter: f => f.startsWith('dc-') }
         - { src: 'rules', dest: 'rules', filter: f => f.startsWith('dc-') }
         - { src: 'templates', dest: 'templates' } (all files)
         - { src: 'tools', dest: 'tools', filter: f => f.startsWith('validate-') }
       - PKG_ROOT: `path.resolve(__dirname, '..')` (INST-07)

    2. **Pure functions (exported for testing):**
       - `parseArgs(argv)` -> { isLocal, isUninstall }
       - `isDcHook(entry)` -> boolean (matches `dc-` in any hook command string)
       - `mergeHooks(settings, dcEntries)` -> mutated settings (filter-then-append)
       - `getDcHookEntries(targetDir, isLocal)` -> { SessionStart: [...], PostToolUse: [...] }
       - `getTargetDir(isLocal)` -> resolved path string

    3. **Side-effect functions:**
       - `copyFiles(targetDir)` -> copies INSTALL_MAP dirs from PKG_ROOT to targetDir
         - Use fs.mkdirSync({ recursive: true }) for target dirs
         - For entries with filter: read source dir, copy only matching files
         - For entries without filter: use fs.cpSync({ recursive: true })
         - After copy, chmod 755 any .sh files
       - `updateSettings(settingsPath, dcEntries)` -> read-modify-write settings.json
         - If file exists, parse it (try-catch; on error, back up to .bak and warn)
         - If file does not exist, start with {}
         - Call mergeHooks
         - Write with JSON.stringify(obj, null, 2) + '\n'

    4. **Main function:**
       - Parse args
       - If --uninstall: delegate to plan 02 (print "uninstall not yet implemented" for now)
       - Determine targetDir
       - Call copyFiles(targetDir)
       - Call updateSettings with getDcHookEntries
       - Print success message placeholder (plan 02 polishes this)

    5. **Module pattern for testability:**
       ```javascript
       // Export for testing
       if (typeof module !== 'undefined') {
         module.exports = { parseArgs, isDcHook, mergeHooks, getDcHookEntries, getTargetDir, INSTALL_MAP };
       }
       // Run if invoked directly
       if (require.main === module) { main(); }
       ```

    Key implementation details per user decisions:
    - Overwrite existing files silently (files are canonical)
    - Mirror source layout under target .claude/
    - Global paths: absolute with quotes for spaces -- `node "/home/user/.claude/hooks/dc-freshness-check.js"`
    - Local paths: relative -- `node .claude/hooks/dc-freshness-check.js`
    - Create settings.json if missing
    - Use `__dirname` (not cwd) to find source files
    - Shebang already present (INST-08), preserve it
  </implementation>
