  <name>Task 1: Create dc-freshness-check.js hook</name>
  <files>hooks/dc-freshness-check.js</files>
  <action>
Create `hooks/dc-freshness-check.js` -- a Node.js SessionStart hook that:

1. **Stdin timeout guard (HOOK-07):** Set a 3-second `setTimeout(() => process.exit(0), 3000)` before reading stdin. Read stdin to satisfy the pipe contract but do not parse its content (SessionStart hooks use `process.cwd()` per `gsd-check-update.js` pattern). Clear timeout on stdin `end`.

2. **Graceful no-op (HOOK-02):** Wrap all logic after stdin handling in try/catch with `process.exit(0)` in catch. Before parsing, check `fs.existsSync(path.join(cwd, '.context', 'MANIFEST.md'))` -- if missing, exit 0 silently.

3. **Parse MANIFEST.md entries (HOOK-01):**
   - Read `.context/MANIFEST.md` via `fs.readFileSync`
   - Split by newline, iterate lines
   - Match entry lines with `/^-\s*\[([^\]]+)\]/` to extract entry name (only lines starting with `- [`)
   - For matched entries, look for `/\[verified:\s*(\d{4})-(\d{2})-(\d{2})\]/` to extract date
   - If entry has NO verified date: add to stale list as `"EntryName (never verified)"` per user decision
   - If entry has verified date: construct `new Date(year, month-1, day)` (local midnight, avoids UTC timezone trap per research pitfall 2), compute days since verified, if >90 days add to stale list as `"EntryName (N days overdue)"` where N = daysSince - 90 per user decision

4. **Output warning via additionalContext:**
   - If no stale entries, exit 0 silently
   - If stale entries exist, build message with:
     - Header: `"Domain Context: N stale entries"` per user decision
     - Indented list of each stale entry (e.g., `"  - Integration Model (35 days overdue)"`)
     - Action line: `"Run /dc:refresh to review and update verified dates."` per user decision
   - Output JSON: `{ hookSpecificOutput: { hookEventName: "SessionStart", additionalContext: message } }` via `process.stdout.write(JSON.stringify(output))`

Use only Node.js built-ins: `fs`, `path`. No external dependencies. Match the coding style of `.claude/hooks/gsd-context-monitor.js` (require statements, const for config values, descriptive comments).

The `THRESHOLD_DAYS` value (90) should be a named constant at the top of the file for easy adjustment.
