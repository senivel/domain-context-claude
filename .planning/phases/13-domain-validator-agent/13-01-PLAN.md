  <action>
Create the directory `agents/` if it does not exist, then create `agents/dc-domain-validator.md` with:

**Frontmatter (YAML):**
- `name: dc-domain-validator`
- `description: Checks project code against business rules documented in .context/domain/ and .context/constraints/ files. Reports structured violation findings.`
- `tools: Read, Grep, Glob` (AGNT-01: read-only tools ONLY -- no Edit, Write, Bash, Agent)
- `color: yellow`

**System prompt body (~80-100 lines, fully self-contained per AGNT-05):**

The prompt must implement a three-phase workflow:

**Phase 1 -- Rule Discovery (AGNT-02):**
1. Read `.context/MANIFEST.md` to discover domain concept and constraint file paths from "Domain Concepts" and "Constraints" sections
2. Read each discovered file from `.context/domain/` directory
3. Read each discovered file from `.context/constraints/` directory (may be empty -- that is fine)
4. Extract numbered items from "Business Rules" and "Invariants" sections (these are the spec-defined sections containing enforceable rules)
5. Classify each rule as code-enforceable or human-judgment-only:
   - Code-enforceable: rules referencing file patterns, naming conventions, structural requirements, import/reference patterns
   - Human-judgment-only: rules about process, timing, or human intent
6. Skip human-judgment-only rules (note them in the summary as "skipped -- requires human judgment")

**Phase 2 -- Code Scanning (AGNT-03):**
1. Use Glob to find code files in scan directories: `src/`, `app/`, `lib/`, `commands/`, `hooks/`
2. Explicitly SKIP: `.planning/`, `node_modules/`, `templates/`, `.git/`, test fixtures
3. For each code-enforceable rule, use Grep to search for patterns indicating violations
4. Use Read to examine flagged files in context for false-positive filtering
5. Prefer skipping borderline cases over reporting false positives

**Phase 3 -- Report (AGNT-04):**
When violations found, produce:
```
## Domain Validation Report

**Scanned:** {directories scanned}
**Rules checked:** {N} ({M} skipped -- require human judgment)

### Violations

| # | Violation | Location | Rule Source |
|---|-----------|----------|-------------|
| 1 | {description} | {file:line} | {source file, Rule N} |

### Rules Checked

| Rule | Source | Status |
|------|--------|--------|
| {rule summary} | {file} | Checked -- {N} violation(s) / no violations |
| {rule summary} | {file} | Skipped -- requires human judgment |
```

When no violations found, produce:
```
## Domain Validation Report

**Scanned:** {directories scanned}
**Rules checked:** {N} ({M} skipped -- require human judgment)

No violations found.

### Rules Checked
{same table as above}
```

**Critical constraints to embed in the prompt:**
- Report-only: do NOT suggest or make fixes (per locked decision: "validator reports, it does not fix")
- Self-contained: do NOT reference @AGENTS.md, @CLAUDE.md, or assume any conversation history
- No `@` file references in the prompt body -- all instructions inline
- Read-only: never attempt to modify any file
