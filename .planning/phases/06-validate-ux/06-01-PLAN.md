  <action>
Rewrite `commands/dc/validate.md` to add Phase 6 capabilities. The skill file structure remains the same (YAML frontmatter, objective, execution_context, process) but with significant enhancements.

**Frontmatter changes:**
- Update `description` to: "Check structural integrity of domain context. Reports broken links, orphan files, stale entries, and AGENTS.md imports. Offers to fix issues found."
- Update `allowed-tools` to: `[Read, Glob, Write, Edit, AskUserQuestion]`

**Objective update:**
- Add mention of AGENTS.md import checking and interactive fix offers.

**Execution_context updates:**
- REMOVE the line "Do NOT offer to fix issues (that is Phase 6 / VALD-05)."
- ADD documentation for the AGENTS.md import check: explain that AGENTS.md should contain `@.context/MANIFEST.md` and `@ARCHITECTURE.md` as substrings.
- ADD documentation for the fix flow: per-group AskUserQuestion pattern, template path resolution for file creation, MANIFEST.md section detection for orphan registration.
- ADD path-to-section mapping table for orphan registration:
  - `.context/domain/` -> `## Domain Concepts`
  - `.context/decisions/` -> `## Architecture Decisions`
  - `.context/constraints/` -> `## Constraints`
  - `**/CONTEXT.md` (outside .context/) -> `## Module Context Files`
- ADD path-to-template mapping for broken link file creation:
  - Domain Concepts section -> `templates/domain-concept.md`
  - Architecture Decisions section -> `templates/decision.md`
  - Constraints section -> `templates/constraint.md`
- ADD entry format documentation for orphan registration (matching existing MANIFEST.md entry formats from Phase 5's execution_context).
- ADD template resolution note: check `.claude/domain-context/templates/` (local) then `~/.claude/domain-context/templates/` (global), same as dc:init Step 1.

**Process changes:**

Steps 1-5 remain as-is from Phase 5. Add/modify the following:

**New Step 5.5 (renumber as Step 6): AGENTS.md Import Check**
1. Try to read `AGENTS.md` from project root.
2. If absent: record a note "No AGENTS.md found. Run /dc:init to set up project structure." -- not an error, just a note.
3. If present: check if the file content contains the substrings `@.context/MANIFEST.md` and `@ARCHITECTURE.md`.
4. Each missing import recorded as a **warning** (not an error).
5. This becomes the 4th check group in results display.

**Modify Step 6 (renumber as Step 7): Format and Display Results**
Keep the existing grouped format but add these enhancements:
- After each group header that has count > 0, add a plain-language explanation line:
  - Broken Links: "{N} entries in MANIFEST.md point to files that don't exist."
  - Orphan Files: "{N} file(s) in .context/ {is/are} not referenced in MANIFEST.md."
  - Stale Entries: "{N} {entry/entries} {has/have} not been verified in over 90 days."
  - AGENTS.md Imports: "AGENTS.md is missing {N} recommended import(s)."
- Groups with 0 items: show header line only (no explanation line). This is unchanged behavior.
- Add 4th group: `{icon} AGENTS.md Imports ({count})` with items like `- Missing: @.context/MANIFEST.md`
- AGENTS.md warnings use the warning icon (same as stale entries).
- If AGENTS.md not found, show the note as a single item under the group.
- Update summary line to include AGENTS.md warnings in the warning count.
- Clean state (all 4 groups with 0 issues): "All checks passed. {N} entries validated." -- same as Phase 5.

**New Step 8: Fix Flow**
Only execute if any group has count > 0. If clean state, skip entirely.

For each group with issues, in order (Broken Links, Orphan Files, Stale Entries, AGENTS.md Imports):

**Broken Links fix:**
- Use AskUserQuestion with prompt: "Broken Links: {N} entries point to files that don't exist."
- Options: 1. "Remove entries from MANIFEST.md" / 2. "Create missing files from template (recommended)" / 3. "Skip"
- If "Remove": Use Edit to remove each broken link's entry line from `.context/MANIFEST.md`.
- If "Create": For each broken link, determine which MANIFEST.md section it's in, select the matching template (domain-concept.md, decision.md, or constraint.md), resolve the template path (local then global install location), read the template, replace `{placeholder}` tokens with reasonable defaults (name from entry, today's date), write the file to the expected path. Use Write tool.

**Orphan Files fix:**
- Use AskUserQuestion with prompt: "Orphan Files: {N} file(s) not referenced in MANIFEST.md."
- Options: 1. "Register in MANIFEST.md (recommended)" / 2. "Skip"
- If "Register": For each orphan, determine the appropriate MANIFEST.md section based on its path. Derive entry name from filename (kebab-case to Title Case; for decisions, extract ADR number from `NNN-title.md` pattern). Use Edit to insert the entry line before the next `##` header (or at EOF). If section contains `(none yet)`, replace that line with the new entry.
- Entry formats:
  - Domain concepts: `- [{Name}]({relative-path}) -- {placeholder description} [public] [verified: {today}]`
  - Decisions: `- [{NNN}: {Title}]({relative-path}) -- {placeholder description} [verified: {today}]`
  - Constraints: `- [{Name}]({relative-path}) -- {placeholder description} [public] [verified: {today}]`
  - Module context: `- {project-root-relative-path} [verified: {today}]`

**Stale Entries fix:**
- Use AskUserQuestion with prompt: "Stale Entries: {N} entries are overdue for review."
- Options: 1. "Update verified dates to today (recommended)" / 2. "Skip"
- If "Update": For each stale entry, use Edit to find and replace the `[verified: {old-date}]` pattern with `[verified: {today}]` in MANIFEST.md. Also read the corresponding context file and update its verified date (look for `[verified: YYYY-MM-DD]` pattern in the file content and replace with today's date).

**AGENTS.md Imports fix:**
- Only offer if AGENTS.md exists and has missing imports (not if AGENTS.md is absent).
- Use AskUserQuestion with prompt: "AGENTS.md Imports: {N} recommended import(s) missing."
- Options: 1. "Add missing imports to AGENTS.md (recommended)" / 2. "Skip"
- If "Add": Read AGENTS.md. If `<!-- domain-context:start -->` sentinel exists, insert missing lines inside that block (after `## Project Context` line or at end of block before `<!-- domain-context:end -->`). If no sentinel, append missing lines at end of file.
- Import line formats: `- Architecture overview: @ARCHITECTURE.md` and/or `- Domain & business context: @.context/MANIFEST.md`

**New Step 9: Post-Fix Summary**
After all fix groups processed, if any fixes were applied:
- Display summary of changes (e.g., "Removed 2 entries from MANIFEST.md.", "Created 1 file from template.", "Registered 1 orphan in MANIFEST.md.", "Updated 3 verified dates.", "Added 1 import to AGENTS.md.").
- End with: "Run /dc:validate again to confirm."
- Do NOT automatically re-run validation.
If no fixes were applied (user skipped all), do not show the post-fix summary.
