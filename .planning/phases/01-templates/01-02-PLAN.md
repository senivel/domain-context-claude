Create tools/ directory and tools/validate-templates.sh -- a bash script that validates all 8 template files.

The script should:

1. **Check file existence (TMPL-01):** Verify all 8 files exist in templates/:
   manifest.md, context.md, domain-concept.md, decision.md, constraint.md, agents-snippet.md, architecture.md, claude.md

2. **Check required sections per file (TMPL-02):** For each template, grep for its required section headings. Use exact heading text from the spec:

   - manifest.md: "# Context Manifest", "## Access Levels", "## Domain Concepts", "## Architecture Decisions", "## Constraints", "## Module Context Files", "## If Restricted Context Is Unavailable"
   - context.md: "## What This Module Does", "## Domain Concepts Owned", "## Business Rules", "## Non-Obvious Decisions", "## What This Module Does NOT Do", "## Dependencies"
   - domain-concept.md: "## What This Is", "## Lifecycle", "## Key Attributes", "## Business Rules", "## Invariants", "## Related Concepts"
   - decision.md: "## Status", "## Context", "## Decision", "## Rationale", "## Consequences", "## Affected Modules"
   - constraint.md: "## Source", "## Requirements", "## Impact on Code", "## Verification"
   - agents-snippet.md: "<!-- domain-context:start -->", "## Project Context", "## Confidential Context", "<!-- domain-context:end -->"
   - architecture.md: "## System Purpose", "## Module Map", "## Data Flow", "## Key Boundaries", "## Technology Decisions"
   - claude.md: "@AGENTS.md"

3. **Check placeholder pattern (TMPL-03):** For files that should have placeholders (manifest, context, domain-concept, decision, constraint, architecture), verify at least one `{snake_case_name}` pattern exists. For files that should NOT have placeholders (agents-snippet, claude), verify NO `{...}` patterns exist.

4. **Check verified date pattern:** For entry templates (context, domain-concept, decision, constraint), verify `<!-- verified: {verified_date} -->` exists.

5. **Check no double curly braces:** Verify no `{{...}}` patterns in any template.

Output format: Print each check with PASS/FAIL. Exit 0 if all pass, exit 1 if any fail. Use color codes for terminal output (green PASS, red FAIL).

After creating the script, make it executable with `chmod +x` and run it. Fix any template issues found.
