Modify the existing `commands/dc/explore.md` skill file to add filesystem discovery of CONTEXT.md files. The changes are:

1. **Add a new Step 3.5 (between current Step 3 and Step 4)** titled "Discover Module Context Files":
   - Use Glob to find all CONTEXT.md files in the codebase: `**/CONTEXT.md`
   - Exclude `.context/` directory results (those are the manifest itself, not module context files)
   - Exclude common non-project directories: `node_modules`, `.git`, `.planning`
   - Cross-reference discovered files against entries already parsed from MANIFEST.md's "Module Context Files" section in Step 3
   - For each discovered file NOT in MANIFEST.md: add it to the Module Context Files list with a `[not in manifest]` tag instead of a verified date

2. **Update Step 4 (Display Summary)** to reflect that Module Context Files now includes both MANIFEST.md entries and discovered-but-unregistered files:
   - The count in `Module Context Files (N)` reflects the total (registered + unregistered)
   - Unregistered entries display with `[not in manifest]` tag
   - MANIFEST entries with missing files display with `[file missing]` tag
   - Example:
     ```
     Module Context Files (3)
       - src/auth/CONTEXT.md            [verified: 2026-03-11]
       - src/api/CONTEXT.md             [file missing]
       - src/billing/CONTEXT.md         [not in manifest]
     ```

3. **Update Step 5 (Progressive Disclosure)** so that Module Context Files entries (including discovered ones) are browseable in the drill-in flow. Unregistered files are still readable -- resolve their path relative to project root (not .context/ directory). For `[file missing]` entries, if the user selects one, display "File not found at {path}" instead of trying to read it.

4. **Update Step 6 (Keyword Search)** to include discovered-but-unregistered CONTEXT.md files in the keyword search scope. Search their file content the same way as registered entries.
