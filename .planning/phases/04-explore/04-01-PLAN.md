  <action>
Create `commands/dc/explore.md` following the exact skill file structure from `commands/dc/init.md`.

**Frontmatter:**
```yaml
name: dc:explore
description: Browse and search domain context in the current project. Shows summary of all entries with freshness status, supports keyword search, and lets you drill into specific entries.
allowed-tools:
  - Read
  - Glob
  - Bash
  - AskUserQuestion
```

**Objective section:** Read-only browsing and searching of .context/ domain knowledge.

**Execution context section:** Describe that MANIFEST.md is at `.context/MANIFEST.md`, explain the entry line formats (linked entries, decision entries, module context entries), and note that empty sections contain parenthetical text like `(none yet)`.

**Process section — implement these steps:**

**Step 1: Check for Arguments**
- If the user provided a keyword argument (e.g., `/dc:explore bridge`), go to Step 5 (Keyword Search).
- Otherwise, continue to Step 2.

**Step 2: Check for .context/ Directory**
- If `.context/` does not exist, display: "No .context/ directory found. Run /dc:init to set up domain context." Then stop.
- If `.context/MANIFEST.md` does not exist, display: "No .context/MANIFEST.md found. Run /dc:init to set up domain context." Then stop.

**Step 3: Parse MANIFEST.md and Display Summary**
- Read `.context/MANIFEST.md`.
- Parse entries under each section header: `## Domain Concepts`, `## Architecture Decisions`, `## Constraints`, `## Module Context Files`.
- For each entry, extract name and verified date. Lines starting with `- [` are linked entries. Lines starting with `- ` followed by a path are module context entries. Lines with parenthetical text like `(none yet)` are NOT entries — skip them.
- For each entry, calculate freshness: if verified date is more than 90 days ago from today, display `[STALE - {N} days]` instead of `[verified: YYYY-MM-DD]`.
- Display summary in this exact format:
```
Domain Context:

  Domain Concepts ({count})
    - {Entry Name}                {freshness}
    - {Entry Name}                {freshness}

  Architecture Decisions ({count})
    - {Entry Name}                {freshness}

  Constraints ({count})

  Module Context Files ({count})
    - {path}                      {freshness}
```
Empty sections show count (0) with no entries listed. Names and freshness are aligned with consistent spacing. No descriptions shown in summary.

**Step 4: Progressive Disclosure (Drill-In)**
- After showing the summary, ask the user via AskUserQuestion: "Explore an entry?"
- First level options (pick section type): "Domain Concepts" / "Architecture Decisions" / "Constraints" / "Module Context Files" / "Done"
  - If there are 5 options (4 sections + Done), combine the last non-empty section with Done or use the recommended grouping that stays within AskUserQuestion's practical limit.
  - Skip sections with 0 entries from the options.
- When user picks a section, show entries within that section as AskUserQuestion options:
  - If 1-2 entries: show entries + "Back" + "Done"
  - If 3+ entries: show 2 entries + "More" + "Back" (paginate)
- When user picks an entry: Read the entry's file (resolve path relative to `.context/`) and display its full content. Show the file path above the content.
- After displaying content, loop back: "Explore another entry?" with the same section/entry selection flow.
- When user picks "Done" at any level, stop.

**Step 5: Keyword Search**
- Read `.context/MANIFEST.md` and parse all entries (same as Step 3).
- For each entry, check if the keyword matches (case-insensitive) in:
  1. Entry name
  2. Entry description (from MANIFEST.md line)
  3. Entry file content (read each entry's file and search)
- If no matches: display "No entries matching '{keyword}' found. Run /dc:explore to see all entries." Then stop.
- If exactly one match: read and display that entry's full file content with file path.
- If multiple matches: list matching entry names with where the match was found (name/description/content). Ask the user via AskUserQuestion which entry to view. Display selected entry's full content.
- After displaying, offer to search again or explore more.

**Important details per CONTEXT.md locked decisions:**
- No fuzzy matching or "did you mean" suggestions
- Case-insensitive matching only
- Module CONTEXT.md files come from MANIFEST.md only — no filesystem glob/discovery
- No descriptions in the summary view — names and freshness only
- Always show summary first, even if there's nothing stale
